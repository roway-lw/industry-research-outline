# Style And Template Rules

## Template Source

Use `assets/临西轴承产业大脑调研大纲V1.1.docx` as the style reference for formal DOCX output.

The template is a formatting and writing-style reference. Do not blindly copy its industry-specific content unless the target industry is also the Linxi bearing industry.

For formal DOCX output, this is not a soft preference. The output must be template-driven. Read `docx-output-workflow.md` before creating the file.

For Word deliverables, also read `docx-format-constraints.md`. Its rules are hard constraints: preserve the template formatting first, then add first-line indentation only to body/question paragraphs.

## Writing Style

Use a practical fieldwork style:

- Write questions that offline researchers can ask directly.
- Prefer concrete industry-chain, data, system, policy, and service questions.
- Avoid long generic background prose.
- Avoid empty terms such as “全面赋能”, “打造标杆”, “助力高质量发展” unless tied to specific research tasks.
- Keep headings formal and concise.
- Use numbered questions under each object type.
- 正文段落、问题组和问题项首行缩进两个字符；标题不做首行缩进。

## Tone

The document should feel like a professional project research outline prepared for a government or park customer. It should be serious, clear, and operational.

## Structure Style

Follow the observed structure of historical outlines:

- Title: `XX产业大脑调研大纲`
- 一、调研背景及目标
- （一）调研背景
- （二）调研目标
- 二、调研对象及问题清单
- （一）政府部门
- （二）园区运营方与行业协会/产业联盟
- （三）重点企业
- （四）行业服务平台

Add extra sections only when they help field execution, such as materials collection, implementation notes, and verification lists.

## DOCX Generation

When creating a DOCX:

- Use the document-generation skill.
- Do not start from a blank DOCX.
- Copy or derive the output from `assets/临西轴承产业大脑调研大纲V1.1.docx`.
- Preserve template styles, margins, fonts, line spacing, title style, heading styles, and numbering feel.
- Insert new content with existing template styles rather than ad-hoc direct formatting.
- Reuse the template's real paragraph and run formatting; do not rely only on generic style names if formatting changes.
- Treat first-line indentation as an additive body-text rule, not as a replacement for the template's original formatting.
- Render and visually inspect the DOCX if the environment supports it.
- Deliver only the final DOCX and requested attachment files unless the user asks for QA intermediates.
