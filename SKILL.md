# 产业大脑调研大纲生成

## 能力定位

使用本 Skill 帮助用户生成产业大脑项目前期使用的、可指导线下团队实地沟通的调研大纲。当前支持两条路线：

1. **区域产业整体调研**：面向一个区域和一个产业，了解产业基础、产业链结构、数据资源、调研对象和产业大脑场景机会。
2. **单一单位合作调研**：面向具体企业或生态机构，判断该单位与产业大脑之间的数据共享、系统对接、服务共建、场景试点、资源导入等合作点。

输出内容应像正式咨询交付物，而不是泛泛的 AI 文章。它要帮助线下调研团队明确拜访谁、问什么、收集什么资料，以及这些信息如何支撑后续产业大脑方案设计。

## 友好入口

每次开始时，先用中文简要介绍能力，然后让用户选择调研类型：

1. 区域产业整体调研大纲
2. 具体单位/机构业务调研大纲

即使用户已经输入了“临西轴承产业”这类清晰表述，也要先识别区域和产业，再让用户通过固定入口选择调研类型。不要直接跳成“请确认是否正确”。完整交互流程见 `references/interaction.md`。

提问必须分阶段进行。首次回复不要一次性抛出长问卷；每轮只问下一步真正需要的 1-2 个关键问题。

## 核心流程

1. **判断调研类型**
   - 如果是区域产业整体调研，走产业整体调研路线。
   - 如果是具体企业、机构、平台、协会、园区、服务商等，走单一单位合作调研路线，并先区分“企业类”还是“生态机构类”。
   - 即使用户输入看起来已经是明确的区域加产业，也必须通过固定的两类入口确认。

2. **分阶段收集输入**
   - 先确认调研类型、区域、产业，或具体单位名称和单位类型。
   - 再询问用户是否已有资料，可以上传文件，也可以提供本地目录路径。
   - 如果资料不足，再询问是否需要检索最新公开资料。
   - 最后询问输出形式：Markdown 大纲草稿、正式 Word 调研大纲，或 Word 大纲加附件包。

3. **优先处理用户资料**
   - 如果用户提供文件或目录，先分析用户资料，再考虑公开检索。
   - 识别已经明确的信息、仍然缺失的信息、需要现场核实的问题。
   - 如果资料不足，询问是否补充检索最新公开资料。不要向用户暴露内部检索工具名称。

4. **必要时使用最新公开资料**
   - 使用当前环境中可用的最佳公开资料检索能力。
   - 内部优先使用已经可用的 AnySearch MCP/工具接口。
   - 不要求用户安装工具、执行检索脚本、授权技术命令或提供 API Key。
   - 优先使用近期、权威、可追溯的资料。
   - 将公开资料视为初步证据，不直接当作最终结论。
   - 检索前阅读 `references/search-strategy.md`。

5. **生成调研交付内容**
   - 区域产业整体调研：阅读 `references/industry-outline-method.md` 和 `references/attachments.md`。
   - 单一单位合作调研：阅读 `references/organization-research-method.md`，根据单位类型生成企业类结构或生态机构类结构。
   - 如果用户需要可下载交付包，创建 `delivery_spec.json`，运行 `scripts/build_delivery_package.py` 生成 DOCX、XLSX 附件、manifest 和 ZIP 包。
   - 生成交付包前阅读 `references/delivery-package.md` 和 `references/delivery-spec-schema.md`。

6. **应用格式规范**
   - Markdown 草稿遵循临西轴承样例的正式写法。
   - 正式 DOCX 不得从空白文档创建，必须使用 `assets/临西轴承产业大脑调研大纲V1.1.docx` 作为模板基础。
   - 生成任何正式 DOCX 前，必须阅读 `references/docx-format-constraints.md`，并将其作为不可放松的硬约束。
   - 正式写作和文件生成前阅读 `references/style-and-template.md`。
   - 创建 DOCX 前阅读 `references/docx-output-workflow.md`。

## 输出要求

区域产业整体调研大纲通常包括：

- 调研背景及目标
- 调研范围与重点
- 产业初步认知
- 调研对象及问题清单
- 分对象访谈提纲
- 资料收集清单
- 产业大脑场景挖掘重点
- 待现场核实问题

区域产业整体调研默认附件包包括：

- 调研对象信息登记表
- 访谈纪要记录表
- 资料收集台账
- 企业调研信息表
- 产业链环节核实表
- 数据资源盘点表
- 产业大脑场景线索表
- 问题假设验证清单

单一单位合作调研大纲必须按单位类型选择结构：

- **企业类**：交流背景及目标、企业基本情况、企业经营与业务现状、核心痛点与需求、产业大脑合作方向、重点交流问题清单、需补充资料清单。
- **生态机构类**：交流背景及目标、机构基本情况、机构资源与服务能力、现有业务与运营模式、产业大脑合作方向、重点交流问题清单、需补充资料清单。

单一单位合作调研默认附件包包括：

- 单位基础信息记录表
- 访谈纪要记录表
- 合作机会梳理表
- 数据资源与系统对接盘点表
- 资料收集与行动清单

## 质量规则

- 默认使用中文输出，除非用户明确要求其他语言。
- 避免空泛口号和通用套话。
- 问题要具体到线下调研人员可以直接问。
- 区分已确认事实、公开资料推断和现场待核实问题。
- 使用公开检索时，尽量保留资料时间和来源日期。
- 公开检索、文件生成、写入磁盘前，需要先获得用户确认。
- 创建 DOCX 交付物时，必须走临西模板驱动流程；如果环境支持渲染，应进行页面视觉检查。
- 用户需要文件时，优先使用交付包脚本，不要临时手工拼文件。
